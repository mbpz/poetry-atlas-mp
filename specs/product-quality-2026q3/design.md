# 产品质量提升设计

## 设计规格

1. **Purpose Statement**：为诗词爱好者提供可信、可探索、可收藏的地理化阅读体验。此次设计重点是让首次使用、权限异常、弱网和回访场景都具备明确反馈与恢复路径。
2. **Aesthetic Direction**：Editorial/magazine（古籍编辑式信息层级），延续现有宣纸淡墨视觉。
3. **Color Palette**：宣纸 `#f6f1e4`、纸面 `#fffef9`、墨色 `#3a3632`、朱砂 `#9e2b23`、描金 `#a88848`。
4. **Typography**：标题与诗文优先 `Songti SC` / `STSong`，短标签优先 `Kaiti SC` / `KaiTi`，必要时以 `PingFang SC` 作为设备兼容回退。该字体栈是现有品牌约束的窄范围保留。
5. **Layout Strategy**：地图作为全屏内容画布，信息以非对称边缘工具组和可收起底部卷页承载；列表页以时间轴、引文和章节节奏组织，避免统一居中卡片堆叠。底部导航保持固定五入口并完整处理安全区。

## 1. 定位韧性

引入单一定位服务边界，页面不再直接解释 `wx.getLocation` 的所有错误。

```text
用户点击定位
  -> 检查应用授权
  -> 请求 GCJ-02 定位
  -> 分类结果
       success          -> 移动地图并标记当前位置
       permission       -> 打开设置并返回后重试
       system-disabled  -> 引导开启系统定位
       precision/ROM    -> 提示精确定位或微信定位权限
       timeout/network  -> 提供重试与手动选城
       unknown          -> 保留错误码并安全降级
```

建议模块：

- `utils/location.js`：状态机、错误归类、重试策略与非敏感诊断事件。
- 地图页：只消费 `locating/success/recoverable/denied/degraded` 状态。
- 手动降级：城市/区域选择、默认全国视野、最近一次地图中心；不将定位设为使用地图的前置条件。
- 测试：错误分类单元测试 + Android 真机矩阵清单。

## 2. 诗词语料治理

数据链路拆为四段：原始来源、规范化、质量门禁、版本发布。

### 规范记录

每首诗词至少包含：

- `canonical_id`：稳定标识，不再只依赖标题和作者。
- `title/author/dynasty/content`：规范展示字段。
- `content_kind`：`full` 或 `excerpt`。
- `source_name/source_url/source_license`：来源与授权信息。
- `content_hash/data_version/review_status`：变更追踪字段。
- `places/place_names`：地点关联，不在地点副本中维护另一份不可控正文。

### 冲突策略

1. 先按规范 ID 合并地点关系。
2. 正文完全一致时去重。
3. 正文为前缀关系时，将短版本标为节选，全文候选进入校订。
4. 正文实质冲突时阻止发布，要求人工选择来源明确的规范版本。
5. `hot_poems` 只嵌入 ID 和展示摘要，避免正文多副本漂移。

### 发布策略

- 审计工具生成 JSON 与 Markdown 报告，并以非零退出码阻止不合格语料发布。
- 先校订高频/高曝光作品，再扩展到全量 491 首。
- 数据发布使用版本清单、前后计数与抽样核验；保留上一版本以支持回滚。

## 3. 成熟产品交互

### 导航外壳

- 固定五入口：地图、发现、朝代、收藏、我的。
- 统一安全区、点击热区、选中态和失败回退；移除与底部导航重复的悬浮搜索入口或明确其唯一职责。
- 解决现有 #1 浮层重叠与 #2 TabBar 缺失问题，并纳入设备验收。

### 三条核心路径

1. 地图发现：地图 -> 定位/手动选城 -> 地点 -> 诗词。
2. 搜索阅读：关键词/热门词 -> 分类结果 -> 诗词/作者/地点 -> 收藏。
3. 私人学习：收藏/个人资料 -> 私人路线创建 -> 路线详情 -> 地点或诗词。

每条路径必须覆盖加载、空数据、错误、重试、返回和重复操作保护。状态组件使用一致的古籍编辑式文案与视觉，不用 Emoji 充当产品图标；图标资产保持单一线性 SVG 语言。

## 4. 质量与可观测性

- 静态层：`node --check`、语料审计、配置一致性检查。
- 逻辑层：定位状态机、正文冲突策略、关键页面状态转换测试。
- 运行层：微信开发者工具预览与 Android/iOS 真机冒烟。
- 指标层：记录功能、阶段和归类错误码；禁止采集精确经纬度、正文输入或用户身份信息。
- 发布层：每个候选版本附设备矩阵、核心路径结果、已知风险和回滚版本。
